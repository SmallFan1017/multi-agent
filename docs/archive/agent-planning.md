# 🎓 研究生 AI Agent 系統規劃

## 一、系統概述

建立一個主 Agent（研究生角色），它能理解複雜的研究任務，並自動拆解、分配給底下的 Sub-Agent 來執行。

### 架構圖

```
┌─────────────────────────────────────────┐
│         主 Agent（研究生 Researcher）       │
│   - 任務理解與拆解                          │
│   - Sub-Agent 調度                         │
│   - 結果整合與品質檢查                       │
└─────────────┬───────────────────────────┘
              │ 呼叫 / 創建
    ┌─────────┼─────────┬──────────┐
    ▼         ▼         ▼          ▼
┌────────┐┌────────┐┌────────┐┌────────┐
│翻譯論文 ││論文整理 ││文獻搜尋 ││ ...更多 │
│Agent   ││Agent   ││Agent   ││Sub-Agent│
└────────┘└────────┘└────────┘└────────┘
```

## Output Format

Markdown 內容需包含：

- 論文標題（如有），以 Markdown 標記（# 開頭）。
- 作者、日期等基本資訊（如有），可翻譯後保留於最開頭。
- 章節標題與子標題應以 #、##、### 等 Markdown 標記分層。
- 段落與一般文字以標準 Markdown 排版處理，務必使用正確語法換行，使內容更易閱讀。
- 【數學公式處理規則】：
  - 行內公式：使用 $...$。
  - 獨立公式（Display Math）：
    1. 必須獨立成段：公式區塊的前後必須各有一行空行。
    2. 格式規範：使用 

       $$
       包圍。**請務必在
       $$

        符號後立即換行，並在公式結束後、結尾 $$ 符號前換行。**
    3. 若原論文中有公式編號（如 (1), (2)），務必使用 \tag{編號} 語法將編號寫在 

       $$
       內部，嚴禁寫在
       $$

        結尾符號之後。
    4. 範例：
       正確：

       $$
       \text{idf}(w) = -\log \frac{1}{M} \sum_{i=1}^M I[w \in x^{(i)}]
       $$

       錯誤：
       $$
       \text{idf}(w) = -\log \frac{1}{M} \sum_{i=1}^M I[w \in x^{(i)}]
       $$
- 【重要：程式寫入安全規範】：
  - 反斜線處理：在呼叫 write_file 或 replace 工具時，Markdown 字串中的 LaTeX 反斜線 \ 必須進行雙重跳脫 (Double Escaping)（即在程式碼字串中寫成 \\，例如 \\alpha），以確保最終寫入檔案的內容保留單一反斜線。
  - 檢查：請確保 \text, \frac, \tag 等指令在寫入檔案時不會變成 text, frac, tag。
- 無法翻譯或格式錯誤的內容請直接略過並加註解。
- 遇到無法解析的 LaTeX 語法，請加註解。

若未輸入論文內容，請回傳：{"translated_md": "未輸入論文內容，無法進行翻譯。"}
