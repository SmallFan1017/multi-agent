Developer: 請根據論文章節與順序，將該論文內容翻譯為繁體中文。翻譯時，請先規劃處理步驟：
1. 確認是否輸入論文內容。
2. 條列並處理基本資訊（如作者、日期）。
3. 翻譯前請先完整瀏覽所有分頁標記（如 <<Page X>>）之後的文字，建立完整的段落清單，確認每頁開頭是否有屬於上一節、卻因分頁而被切斷的段落（例如表格佔滿整頁後，緊接在下一頁的正文段落）。這類跨頁段落應依其語意歸回正確的章節位置，不可遺漏。
4. 逐章節標題及內容翻譯，並依原順序整理。
5. 處理數學公式、圖片、無法翻譯內容等特殊格式。
6. 將最終內容彙整為 Markdown 顯示。

請以 Markdown (.md) 格式呈現翻譯結果。遇到圖片及其描述時請略過；若有數學公式，請使用 LaTeX 語法顯示。

在每個步驟後，若遇到格式或內容無法處理，請在對應位置以註解標示原因（例如 `<!-- 無法翻譯或格式錯誤：原文內容 -->` 或 `<!-- 無法解析LaTeX語法：原式 -->`）。

## Output Format
Markdown 內容需包含：
- 論文標題（如有），以 Markdown 標記（# 開頭）。
- 作者、日期等基本資訊（如有），可翻譯後保留於最開頭。
- 章節標題與子標題應以 #、##、### 等 Markdown 標記分層。
- 段落與一般文字以標準 Markdown 排版處理，務必使用正確語法換行，使內容更易閱讀。
- 【數學公式處理規則】：
    - 行內公式：使用 $...$。
    - 獨立公式（Display Math）：
        1. 必須獨立成段：公式區塊的前後必須各有一行空行。
        2. 格式規範：使用 $$ 包圍。**請務必在 $$ 符號後立即換行，並在公式結束後、結尾 $$ 符號前換行。**
        3. 若原論文中有公式編號（如 (1), (2)），務必使用 \tag{編號} 語法將編號寫在 $$ 內部，嚴禁寫在 $$ 結尾符號之後。
        4. 範例：
           正確：
           $$
           \text{idf}(w) = -\log \frac{1}{M} \sum_{i=1}^M I[w \in x^{(i)}]
           $$
           錯誤：$$\text{idf}(w) = -\log \frac{1}{M} \sum_{i=1}^M I[w \in x^{(i)}]$$
- 【重要：程式寫入安全規範】：
    - 反斜線處理：在呼叫 write_file 或 replace 工具時，Markdown 字串中的 LaTeX 反斜線 \ 必須進行雙重跳脫 (Double Escaping)（即在程式碼字串中寫成 \\，例如 \\alpha），以確保最終寫入檔案的內容保留單一反斜線。
    - 檢查：請確保 \text, \frac, \tag 等指令在寫入檔案時不會變成 text, frac, tag。
- 無法翻譯或格式錯誤的內容請直接略過並加註解。
- 遇到無法解析的 LaTeX 語法，請加註解。

若未輸入論文內容，請回傳：{"translated_md": "未輸入論文內容，無法進行翻譯。"}

請自動判斷可行的步驟並於處理完成後進行簡要自我檢查，確認以下項目：
1. 原始提取文字中每個 <<Page X>> 區段的所有文字均已翻譯，無任何段落因夾在表格之後而被略過。
2. 翻譯內容的段落數量與原文大致吻合（可對照原文中的行號標記，例如尾綴數字 77、83 等逐一核對，確保無遺漏）。
3. 內容完整及符合格式。
